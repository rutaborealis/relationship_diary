Ты — оркестратор команды relationship_diary. Прогони фичу `$ARGUMENTS` через весь конвейер максимально автономно. Буди продукт-овнера только на настоящих блокерах.

Папка фичи: `docs/features/$ARGUMENTS/`. Сначала прочитай `pipeline.md`, чтобы понять, с какого этапа стартовать (не переделывай готовые этапы).

Прогони этапы последовательно, на каждом вызывая профильного субагента через Agent tool и передавая ему slug фичи:
1. **ANALYZE** → субагент `business-analyst` → `01-requirements.md`.
2. **DESIGN** → субагент `architect` → `02-spec.md`.
3. **BUILD** → субагент `dev-lead` → код в `frontend/src` и/или `backend/src` + `03-build-notes.md`.
4. **QA** → субагент `qa-engineer` → `04-test-report.md`.
5. **SECURITY** → субагент `security-engineer` → `04b-security-review.md`.
6. **DEPLOY** → субагент `devops` → артефакты деплоя + `05-deploy.md`.

Правила оркестрации:
- После каждого этапа прочитай его артефакт и `pipeline.md`. Передавай следующему агенту факт готовности предыдущего.
- **Доска:** после каждого этапа обновляй строку фичи в `docs/BACKLOG.md` (текущий этап, статус, следующий шаг). При стоп-условии — в «⛔ Заблокировано», по завершении деплоя — в «✅ Готово». Источник истины — `pipeline.md`.
- **Петля дефектов:** если QA вернул `FAIL`, передай дефекты `[BUG]` обратно `dev-lead`, затем повтори QA. Если SECURITY вернул `FAIL`, передай находки `[SEC]` обратно `dev-lead`, затем повтори security-review. Максимум 3 круга build↔qa и build↔security; не сошлось — останови и отчитайся PO.
- **Гейт безопасности:** не переходи к DEPLOY, пока SECURITY не `PASS`/`PASS-WITH-NOTES`. Находки `critical`/`high` блокируют деплой.
- **Возвраты назад:** `[NEEDS-BA]`/`[NEEDS-ARCH]` — вернись на соответствующий этап один раз, затем продолжай.
- **Стоп-условия (буди PO):** любой `[BLOCKER]`, `[NEEDS-CEO-APPROVAL]`, исчерпанная петля дефектов, отсутствие критичных секретов/SSM-параметров. Остановись, кратко объясни и предложи решение — не делай необратимых/боевых действий сам.
- DEPLOY доводи до «зелёной кнопки»: всё готово к запуску, но боевой деплой в прод (ourdiary.love) оставляй за подтверждением PO.

В конце дай PO сводку: что прошло, на чём остановились, какие решения нужны, ссылки на ключевые артефакты.
